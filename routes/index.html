Routes

Custom routing and redirects in Mixture.

  1. Introduction
  2. Details

Introduction

Routing and the friendly URL's generated in Mixture are created by default from the structure and filename, but we have now added an additional routing feature to Mixture.

Details

Routes are managed via the mixture settings file, the following is simply an example to demonstrate some options...

"routes": [
  { "route": "blog/page-:page", "template": "blog", "redirect": null },
  { "route": "blog/:month/:year", "template": "blog", "redirect": null },
  { "route": "portfolio/:category", "template": "portfolio", "redirect": null },
  { "route": "search", "template": null, "redirect": "http://google.com" },
  { "route": "blog/:article", "template": null, "redirect": "http://blog.com/{article}" },
  { "route": "blog/:article/:page", "template": null, "redirect": "/articles/{article}/{page}" }
]

The order of routes is important as the first match will win.

You can set variables using :variable and they will become availble to the matching route template via the model like so {{ mixture.route.variable }}

Redirects will report 301 Moved Permanently
