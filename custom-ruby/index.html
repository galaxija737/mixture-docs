Custom Ruby

Point Mixture at a custom Ruby install.

  1. Introduction
  2. Instructions
  3. Windows
  4. Mac

Introduction

If you've been using this feature previously, we've changed the way it works, you no longer need the setting "customRubyPath" so please remove - please read the instructions below...

Mixture uses it's own install of Ruby, Compass and Sass, but if you would like to use your own install and have more control of the Gems, you can point Mixture at a custom install.

Instructions

Simply add or update the following to your Mixture settings "customRuby": true

Mixture will automatically find the necessary paths from the system for your currently active version of Ruby.

Don't forget to ensure you have the necessary gems installed.

If you have any problems or need to add the paths to ruby manually, please read the instructions below for your operating system.

Windows Custom Path

You'll need to update or add (if missing) the following setting to your mixture.json file.

"customRubyPath"

Simply change this to the path of your Ruby install, for example if my install was located c:/ruby200 I would add the following path "customRubyPath": "/Ruby200/".

Note: unlike the Mac version you do not need to add the path to the Gems folder.

Mac Custom Path

You'll need to update or add (if missing) the following setting to your mixture.json file.

Note: The Mac version requires that you also add a path to the Gems.

"customRubyPath"

Simply change this to the path of your Ruby install and Gems, for example if my install was located /Users/xxxx/ruby and my gems /Users/xxxx/gems/ruby I would add the following path "customRubyPath": "/Users/xxxx/ruby|/Users/xxxx/gems/ruby" — notice how the directories are split by a pipe "|".

You can find out the information you need about your install (paths) by running gem env

Here is an example from running gem env to give you an idea:

- RUBY EXECUTABLE: /Users/xxxx/.rvm/rubies/ruby-1.9.1-head/bin/ruby
- EXECUTABLE DIRECTORY: /Users/xxxx/.rvm/gems/ruby-1.9.1-head/bin
- RUBYGEMS PLATFORMS:
- ruby
- x86-darwin-11
- GEM PATH:
- /Users/xxxx/.rvm/gems/ruby-1.9.1-head
- /Users/xxxx/.rvm/gems/ruby-1.9.1-head@global

The first bit we need is the RUBY EXECUTABLE without /bin/ruby eg: /Users/petenelson/.rvm/rubies/ruby-1.9.1-head

Then append this into the customRubyPath with GEM PATH separated with: |

"customRubyPath": "/Users/petenelson/.rvm/rubies/ruby-1.9.1-head|/Users/petenelson/.rvm/gems/ruby-1.9.1-head"

of course your own paths will often be pretty different from this but if you follow the pattern above you should be good to go.

Please note that you may need to try a few different GEM paths if you have issues.
