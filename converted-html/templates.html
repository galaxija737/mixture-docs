Templates
Mixture projects benefit from powerful templating features.

Introduction

Mixture uses the Liquid template engine, Liquid is simply HTML with some additional tags and features.
If you understand HTML then you're all set to go! the additional functionaility Liquid offers is super easy to pick up.
Liquid is an extraction from the e-commerce system Shopify. It's a small and fast template language which is quick to learn but very powerful.

If you prefer to use straight HTML, Mixture can also handle .html and .htm files.

Mixture also supports Haml.

Structure

Mixture allows you to structure the project how you want.

You can control the name and root location of your templates within the Mixture settings.

Note: The name of each template will become the Url

• Naming a file about.liquid in the root of your template folder will result in the Url /about.

• You can sub-folder your templates information/contact-us.liquid which will result in the Url /information/contact-us.

• Mixture automatically strips special characters out of the file name, so as it is safe to be used as a Url.

You can also use standard HTML files in Mixture. You can navigate HTML pages by using the full URL (including the extension, eg /about.html).

Layout, Includes & Blocks

Using layouts, includes & blocks will enable you to keep your code DRY.

You can control the name and root location of your layouts & includes using the Mixture config.

Tags

  • include includes another template; useful for partials (by default Mixture will look for includes within the template directory under a directory called "includes")
  • block replaceable "sections" within a layout file
  • layout pages inherit a layout file (by default Mixture will look for layouts within the template directory under a directory called "layouts")

Examples:

Template files inherit a layout by default. Mixture will automatically assign the layout file detailed in the Mixture settings. You can override this within a template with either an alternative template or none.

Note: the tag to override a layout must be the first line in your template file.

Example loading an alternate layout:

  1. {% layout "alternative" %}

Example setting no layout:

  1. {% layout "none" %}

Example calling an include from a layout or a template:

  1. {% include "navigation" %}

For an include or a layout you do not enter the extension, but you can use sub-folders:

  1. {% include "home/navigation" %}

Layouts, includes and pages have access to all defined variables (model data, variables assigned on the fly in liquid, etc). But you can explicitly define variables and pass through — passing a variable to a liquid tag within an include (tag matching include name).:

  1. {% include "colour" with "green" %}

Include example for the above

  1. My favourite colour is {{ colour }}

If you use an include within a for loop, the loop variables are passed by default and namespaced with the loop variable, eg...

  1. {% for people in team %}
  2.   {% include "template" %}
  3. {% endfor %}

Template...

  1. {{ people.firstname }} {{ people.lastname }}

Blocks

Blocks allow you to define sections within a layout that the view page can inject content into. If the view page does not have the same block the page will render nothing unless the block within the layout has default content.

Example block within a layout file:

  1. {% block content %}My default content{% endblock %}

Example block within the page (which inherits the layout):

  1. {% block content %}My page content{% endblock %}

You can enter as many blocks as needed.

Custom 404

If you wish to use a custom 404 in your project, it's simple... drop 404.liquid into the root of your template directory and you're good to go.

Output

There are two types of markup in Liquid: Output and Tag.

• Output markup (which may resolve to text) is surrounded by:

{{ matched pairs of curly brackets (ie, braces) }}

• Tag markup (which cannot resolve to text) is surrounded by:

{% matched pairs of curly brackets and percent signs %}

Advanced output: Filters

Output markup takes filters. Filters are simple methods. The first parameter is always the output of the left side of the filter. The return value of the filter will be the new left value when the next filter is run. When there are no more filters, the template will receive the resulting string.

Hello {{ 'tobi' | upcase }} Hello tobi has {{ 'tobi' | size }} letters! Hello {{ '*tobi*' | textilize | upcase }} Hello {{ 'now' | date: "%Y %h" }}

Standard Filters

  • date reformat a date
  • capitalize capitalize the first word in the input sentence
  • camelize or camelcase capitalize each word in the input sentence (spaces & characters removed, but hipens converted to spaces)
  • downcase convert an input string to lowercase
  • upcase convert an input string to uppercase
  • first get the first element of the passed in array
  • last get the last element of the passed in array
  • join join elements of the array with certain character between them
  • sort sort elements of the array
  • map map/collect an array on a given property
  • size return the size of an array or string
  • escape escape a string
  • escape_once returns an escaped version of html without affecting existing escaped entities
  • uri_encode URL encodes a string
  • uri_decode URL decodes a string
  • strip_html strip html from string
  • strip_newlines strip all newlines (\n) from string
  • newline_to_br replace each newline (\n) with html break
  • replace replace each occurrence e.g. {{ 'foofoo' | replace:'foo','bar' }} = 'barbar'
  • replace_first replace the first occurrence e.g. {{ 'barbar' | replace_first:'bar','foo' }} = 'foobar'
  • remove remove each occurrence e.g. {{ 'foobarfoobar' | remove:'foo' }} = 'barbar'
  • remove_first remove the first occurrence e.g. {{ 'barbar' | remove_first:'bar' }} = 'bar'
  • truncate truncate a string down to x characters
  • truncatewords truncate a string down to x words
  • html_encode encode HTML
  • prepend prepend a string e.g. {{ 'bar' | prepend:'foo' }} = 'foobar'
  • append append a string e.g. {{ 'foo' | append:'bar' }} = 'foobar'
  • minus subtraction e.g. {{ 4 | minus:2 }} = 2
  • plus addition e.g. {{ '1' | plus:'1' }} = '11', {{ 1 | plus:1 }} = 2
  • times multiplication e.g {{ 5 | times:4 }} = 20
  • divided_by division e.g. {{ 10 | divided_by:2 }} = 5
  • split split a string on a matching pattern e.g. {{ "a~b" | split:~ }} = ['a','b']
  • modulo remainder, e.g. {{ 3 | modulo:2 }} = 1

The date filter

To publish the current date/time

{{ 'now' | date: "format as detail below" }}
  • %a Abbreviated weekday eg Sun
  • %A Full weekday eg Sunday
  • %b Abbreviated month eg Jan
  • %B Full month eg January
  • %c Preferred local date and time
  • %d Day of the month (01 - 31)
  • %H Hour of the day, 24 hour (00 - 23)
  • %I Hour of the day. 12 hour (01 - 12)
  • %j Day of the year (001 - 366)
  • %m Month of the year (01 - 12)
  • %M Minute of the hour (00 - 59)
  • %p Meridian indicator (AM or PM)
  • %S Second of the minute (00 - 60)
  • %U Week number of the current year, starting Sunday (00 -53)
  • %W Week number of the current year, starting Monday (00 - 53)
  • %w Day of the week (Sunday is 0, 0 - 6)
  • %x Preferred representation for the date, no time
  • %X Preferred representation for the time, no date
  • %y Year without a century (00 - 99)
  • %Y Year with century
  • %Z Time zone name
  • %% Literal % character

Tags

Tags are used for the logic in your template. New tags are very easy to code, so I hope to get many contributions to the standard tag library after releasing this code.

Here is a list of currently supported tags:

  • assign assigns some value to a variable
  • capture block tag that captures text into a variable
  • case block tag, its the standard case...when block
  • comment block tag, comments out the text in the block
  • cycle cycle is usually used within a loop to alternate between values, like colors or DOM classes
  • for for loop
  • continue and break used inside a for loop
  • if standard if/else block
  • raw temporarily disable tag processing to avoid syntax conflicts
  • unless mirror of if statement

Comments

Comment is the simplest tag. It just swallows content.

We made 1 million dollars {% comment %} in losses {% endcomment %} this year

Raw

Raw temporarily disables tag processing. This is useful for generating content (eg, Mustache, Handlebars) which uses conflicting syntax.

In Handlebars, {{ this }} will be HTML-escaped, but {{{ that }}} will not.

If / Else

if / else should be well-known from any other programming language. Liquid allows you to write simple expressions in the if or unless (and optionally, elsif and else) clause:

  1. {% if user %}
  2.   Hello {{ user.name }}
  3. {% endif %}
  4.  
  5. {% if user.name == 'tobi' %}
  6.   Hello tobi
  7. {% elsif user.name == 'bob' %}
  8.   Hello bob
  9. {% endif %}
  10.  
  11. {% if user.name == 'tobi' or user.name == 'bob' %}
  12.   Hello tobi or bob
  13. {% endif %}
  14.  
  15. {% if user.name == 'bob' and user.age > 45 %}
  16.   Hello old bob
  17. {% endif %}
  18.  
  19. {% if user.name != 'tobi' %}
  20.   Hello non-tobi
  21. {% endif %}
  22.  
  23. # Same as above
  24. {% unless user.name == 'tobi' %}
  25.   Hello non-tobi
  26. {% endunless %}
  27.  
  28. # Check if the user has a credit card
  29. {% if user.creditcard != null %}
  30.    poor sob
  31. {% endif %}
  32.  
  33. # Same as above
  34. {% if user.creditcard %}
  35.    poor sob
  36. {% endif %}
  37.  
  38. # Check for an empty array
  39. {% if user.payments == empty %}
  40.    you never paid !
  41. {% endif %}
  42.  
  43. {% if user.age > 18 %}
  44.    Login here
  45. {% else %}
  46.    Sorry, you are too young
  47. {% endif %}
  48.  
  49. # array = 1,2,3
  50. {% if array contains 2 %}
  51.    array includes 2
  52. {% endif %}
  53.  
  54. # string = 'hello world'
  55. {% if string contains 'hello' %}
  56.    string includes 'hello'
  57. {% endif %}

Case Statement

If you need more conditions, you can use the case statement:

  1. {% case condition %}
  2. {% when 1 %}
  3. hit 1
  4. {% when 2 or 3 %}
  5. hit 2 or 3
  6. {% else %}
  7. ... else ...
  8. {% endcase %}

Example:

  1. {% case template %}
  2.  
  3. {% when 'label' %}
  4.      // {{ label.title }}
  5. {% when 'product' %}
  6.      // {{ product.vendor | link_to_vendor }} / {{ product.title }}
  7. {% else %}
  8.      // {{page_title}}
  9. {% endcase %}

Cycle

Often you have to alternate between different colors or similar tasks. Liquid has built-in support for such operations, using the cycle tag.

  1. {% cycle 'one', 'two', 'three' %}
  2. {% cycle 'one', 'two', 'three' %}
  3. {% cycle 'one', 'two', 'three' %}
  4. {% cycle 'one', 'two', 'three' %}
  5.  
  6. will result in
  7.  
  8. one
  9. two
  10. three
  11. one

If no name is supplied for the cycle group, then it's assumed that multiple calls with the same parameters are one group.

If you want to have total control over cycle groups, you can optionally specify the name of the group. This can even be a variable.

  1. {% cycle 'group 1': 'one', 'two', 'three' %}
  2. {% cycle 'group 1': 'one', 'two', 'three' %}
  3. {% cycle 'group 2': 'one', 'two', 'three' %}
  4. {% cycle 'group 2': 'one', 'two', 'three' %}
  5.  
  6. will result in
  7.  
  8. one
  9. two
  10. one
  11. two

For Loops

Liquid allows for loops over collections:

  1. {% for item in array %}
  2.   {{ item }}
  3. {% endfor %}

During every for loop, the following helper variables are available for extra styling needs:

  1. forloop.length      # => length of the entire for loop
  2. forloop.index       # => index of the current iteration
  3. forloop.index0      # => index of the current iteration (zero based)
  4. forloop.rindex      # => how many items are still left?
  5. forloop.rindex0     # => how many items are still left? (zero based)
  6. forloop.first       # => is this the first iteration?
  7. forloop.last        # => is this the last iteration?

There are several attributes you can use to influence which items you receive in your loop

limit:int lets you restrict how many items you get. offset:int lets you start the collection with the nth item.

  1. # array = [1,2,3,4,5,6]
  2. {% for item in array limit:2 offset:2 %}
  3.   {{ item }}
  4. {% endfor %}
  5. # results in 3,4

Reversing the loop

  1. {% for item in collection reversed %} {{item}} {% endfor %}

Instead of looping over an existing collection, you can define a range of numbers to loop through. The range can be defined by both literal and variable numbers:

  1. # if item.quantity is 4...
  2. {% for i in (1..item.quantity) %}
  3.   {{ i }}
  4. {% endfor %}
  5. # results in 1,2,3,4

Variable Assignment

You can store data in your own variables, to be used in output or other tags as desired. The simplest way to create a variable is with the assign tag, which has a pretty straightforward syntax:

  1. {% assign name = 'freestyle' %}
  2.  
  3. {% for t in collections.tags %}{% if t == name %}
  4.   <p>Freestyle!</p>
  5. {% endif %}{% endfor %}

Another way of doing this would be to assign true / false values to the variable:

  1. {% assign freestyle = false %}
  2.  
  3. {% for t in collections.tags %}{% if t == 'freestyle' %}
  4.   {% assign freestyle = true %}
  5. {% endif %}{% endfor %}
  6.  
  7. {% if freestyle %}
  8.   <p>Freestyle!</p>
  9. {% endif %}

If you want to combine a number of strings into a single string and save it to a variable, you can do that with the capture tag. This tag is a block which "captures" whatever is rendered inside it, then assigns the captured value to the given variable instead of rendering it to the screen.

  1. {% capture attribute_name %}{{ item.title | slugify }}-{{ i }}-color{% endcapture %}
  2.  
  3. <label for="{{ attribute_name }}">Color:</label>
  4. <select name="attributes[{{ attribute_name }}]" id="{{ attribute_name }}">
  5.   <option value="red">Red</option>
  6.   <option value="green">Green</option>
  7.   <option value="blue">Blue</option>
  8. </select>

Mixture Specific

Mixture extends the Liquid template engine and introduces it's own tags and filters. We recommend that you familiarise your self with the conventions below as they can aid smoother development within Mixture.

Mixture Filters

  • asset_url returns the full url to an asset locally and online.
  • stylesheet_tag used in conjunction with asset_url will return a full stylesheet tag.
  • script_tag used in conjunction with asset_url will return a full script tag.
  • slugify strips special characters out of the string, so you can use the text in a url.
  • lorem_ipsum a lorem ipsum generator.
  • placeholder generates a placeholder image on the fly.
  • image_tag great for use with placeholder, wraps a string in an image tag.
  • paginate used for paging through array / collections.
  • navigation used to dynamically generate navigation from the mixture.sitemap but can be used for any similar model.
  • collection see collections section.
  • grouped see collections section.
  • archive see collections section.
  • item see collections section.

Asset Url Tag

asset_url is magic! enter as much of the path or even just the filename and Mixture will find the file. The more path you add the more narrow the search, so you can focus if you have images with the same name in different locations. It's important to remeber that case is important locally and for published projects, so if your asset is not returned please check the case.

Example returning a full script tag:

  1. {{ "index.js" | asset_url | script_tag }}

Example returning a full stylesheet tag:

  1. {{ "styles/index.css" | asset_url | stylesheet_tag }}

Example using a standard stylesheet tag:

  1. <link href="{{ "index.css" | asset_url }}" rel="stylesheet" type="text/css" />

Example using a standard image tag:

  1. <img src="{{ "example.jpg" | asset_url }}">

Lorem Ipsum Tag

Example using the lorem ipsum tag (defining 2 paragraphs, but no html markup):

  1. {{ 2 | lorem_ipsum }}

Example using the lorem ipsum tag (defining 5 paragraphs within p tags):

  1. {{ 5 | lorem_ipsum:'p' }}

Placeholder Tag Examples

The placeholder tag will return a string, use with image_tag to return a an image tag

Generating a placeholder image with width + height and text + background colours:

  1. {{ "250x250" | placeholder:'FF6600','000000' }}

Generating a placeholder image with width set and colours:

  1. {{ "250" | placeholder:'FF6600','000000' }}

Generating a placeholder image with only width set:

  1. {{ "250" | placeholder }}

Generating a placeholder image with only width set and background colour only:

  1. {{ "250" | placeholder:'FF6600' }}

Example using placeholder with image tag to generate a full img tag.

  1. {{ "250" | placeholder | image_tag }}

Example using placeholder with image tag and custom alt.

  1. {{ "250" | placeholder:'FF6600' | image_tag:'Placeholder' }}

If you publish to Mixtures servers or convert to HTML the placeholder element is switched to the http://imgsrc.me service (thanks to @olliewells, @hamishtaplin, @_fanka, @h_matt & @rdCarp). Locally Mixture will take care of this so as you don't need a connection to use.

Paginate

Within the paginate block the following variables are available:

  • paginate.items the array of objects to display
  • paginate.pages the total pages
  • paginate.previous true or false
  • paginate.next true or false
  • paginate.size total collection size
  • paginate.current_offset offset of items
  • paginate.current_page the current page number
  • paginate.page_size set page size

The following example demonstrates pagination through a model collection 2 items at a time, you can of course customise the logic and markup:

  1. {% paginate model.collection by 2 %}
  2.  
  3. {% for item in paginate.items %}
  4.   {{ item.name }}
  5. {% endfor %}
  6.  
  7. {% if paginate.previous %}
  8.   {% if paginate.current_page != 2 %}
  9.     {% capture prev_page %}/?page={{ paginate.current_page | minus:1 }}{% endcapture %}
  10.   {% endif %}
  11.   <a href="{{ prev_page }}">Previous</a>
  12. {% endif %}
  13.  
  14. Showing items {{ paginate.current_offset | plus: 1 }}-{% if paginate.next %}{{ paginate.current_offset | plus: paginate.page_size }}{% else %}{{ paginate.size }}{% endif %} of {{ paginate.size }}.
  15.  
  16. {% if paginate.next %}
  17.   {% capture next_page %}/?page={{ paginate.current_page | plus:1 }}{% endcapture %}
  18.   <a href="{{ next_page }}">Next</a>
  19. {% endif %}
  20. 			
  21. {% endpaginate %}

The following is the model used in the demonstration above

  4.       "name": "Neil Kinnish"
  5.     },
  6.     {
  7.       "name": "Pete Nelson"
  8.     },
  9.     {
  10.       "name": "Mike Kus"
  11.     },
  12.     {
  13.       "name": "Oli Lisher"
  14.     },
  15.     {
  16.       "name": "Joe Bloggs"

  19. }

Navigation

Example navigation using the model data mixture.sitemap:

  1. {{ mixture.sitemap | navigation: mixture.url, 'ol', 'li', 'selected', 'Home' }}

Explanation of settings (based on the above example:

  • mixture.sitemap this is the data we are using.
  • mixture.url this is the URL of the page you are on.
  • 'ol' this can be empty, but defines the element to wrap each parent.
  • 'li' this can be empty, but defines the element to wrap each child.
  • 'selected' this can be empty, but defines the class to be applied if the URL of the element matches the passed URL.
  • 'Home' this allows you to override the name of the first element in the list.

For more information on "mixture.sitemap" please read the sitemap section.

Haml

Support for Haml is currently classed as experimental.

The best way to try Haml with Mixture is to create a new project and use the "Haml Example" boilerplate.

To learn more about Haml please view the docs. Below is a brief outline of some Mixture specific implementations.

Layouts

Unlike Liquid the Mixture implementation of Haml pages do not auto inherit a Layout file. The following details explain how to use a layout with Haml:

  • Add your HAML layout file to the layouts directory eg. layout.haml
  • Add a refence to the layout within your page eg. index.haml reference at the top of the file =layout :layout

Here's an example page, you will see that we are also updating content in the layout from the page (in this case the title).

  1. =layout :layout
  2. -content_for :title do
  3.   %title Mixture
  4. %p
  5.   A simple p tag

And here is the layout file

  1. !!!
  2. %html
  3.   %head
  4.     = yield :title
  5.   %body
  6.     = yield

Includes (partials)

Includes should be prefixed with an underscore and should be added to the "includes" directory eg. _example_partial.haml

Calling an include from a page or layout:

  1. =render :example_partial

Example include _example_partial.haml

  1. %p
  2.   A simple p tag.

Asset Url

Example usage

  1. :Asseturl
  2.   %a{:href => "my-file.jpg"}
  1. :Asseturl
  2.   %img{:src => "my-file.jpg"}
