Routes
Custom routing and redirects in Mixture.

Introduction

Routing and the friendly URL's generated in Mixture are created by default from the structure and filename, but we have now added an additional routing feature to Mixture.

Details

Routes are managed via the mixture settings file, the following is simply an example to demonstrate some options...

  1. "routes": [
  2.   { "route": "blog/page-:page", "template": "blog", "redirect": null },
  3.   { "route": "blog/:month/:year", "template": "blog", "redirect": null },
  4.   { "route": "portfolio/:category", "template": "portfolio", "redirect": null },
  5.   { "route": "search", "template": null, "redirect": "http://google.com" },
  6.   { "route": "blog/:article", "template": null, "redirect": "http://blog.com/{article}" },
  7.   { "route": "blog/:article/:page", "template": null, "redirect": "/articles/{article}/{page}" }
  8. ]

The order of routes is important as the first match will win.
