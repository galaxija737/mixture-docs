Introduction

Mixture can also work on non-mixture projects to aid your development, we've called this "simple" mode.

Mixture works in partnership with the server for that environment (mamp, wamp, etc) and super-charges it with Mixture features.

This page details the features that are available.

Demonstration screencasts:

  • Using Mixture with any type of project in Simple mode: Mixture + Wamp demo (Windows).
  • Using Mixture with any type of project in Simple mode: Mixture + Mamp demo (Mac).

You can setup a simple mode project to work with a partner server (mamp, wamp, rails, etc), if you don't set a partner sever Mixture will boot it's own simple server so as you can view local html files and assets.

Server

Mixture projects have access to a builtin server and the features this offers.

Simple mode projects (not using the "simpleModePartnerServer" setting) have access to a lightweight version of this server allowing you to view and parse local assets and html files or alternatively you can tell Mixture about another server the project uses (such as mamp, wamp, rails, etc) and Mixture will partner with that server and enable the live refresh, style injection and multi-device capability automatically.

If you encounter issues starting the server from "view locally" or it is greyed out, please visit our support site and knowldege base.

Preprocessors

Mixture will handle compilation automatically for Sass, Less, Stylus and CoffeeScript. We've also introduced our own type "Mix" which simply extends the functionality of standard script and style files, allowing you to combine in a similar way to using the import functions of Sass or Less.

For the full lowdown on this please view the Preprocessors page.

Minification

By default Mixture will create minified versions of all scripts and stylesheets.

For the full lowdown on this please view the Optimisation page.

Optimisation

Mixture will automatically optimise & compress images added to the project by default (currently this works against Jpeg, Jpg, Gif and Png file types).

For the full lowdown on this please view the Optimisation page.

Live Refresh / Multi-device testing

Mixture will automatically reload or style inject when it detects changes in your project. This feature will work on most browsers and devices and is perfect for multi-device testing.

For this feature to work in simple mode you have two options

1. Tell Mixture the location of your server (rails, mamp, .net, etc...). You can do this by editing simpleModePartnerServer within the mixture.json file eg. "simpleModePartnerServer": "http://localhost:8000"

2. Leave "simpleModePartnerServer" null and let Mixture boot up a lightweight version of it's own server so as you can use with static html files and assets.

For more information on live refresh please see the Mixture section. For more information on adjusting settings, please see the Settings section.

Mac users: If you are using Mixture and pointing at a server that runs on a custom hostname and ends with .local or .dev this can sometimes cause a slowdown and is a known issue with OSX since lion. We are looking at workarounds, but if you use localhost or a different name it will be fast again!

Smart To-dos

A unique feature of Mixture is the ability to add to-dos in your code, to-dos are essentially comments that Mixture will find and list for easy access.

For the full lowdown on this please view the Mixture page.
