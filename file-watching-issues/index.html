File Watching Issues

A quick explanation as to why Mixture might be having issues watching a project.

  1. Details

Details

Mixture relies heavily on the file notification API of the OS (OSX, Windows 7, 8 etc) on both the Mac and Windows application.

When a file is changed, deleted, renamed or updated the OS notifies and Mixture then carries out any file watching logic - for example: Sass preprocessing.

If you are using Mixture to watch a project located on a network drive which doesn't have the necessary OS to notify of file changes then Mixture will never be notified and none of the file watching logic within Mixture will be fired.

Some file watching services can also block file watching services like Mixture. A good example of this is Dropbox, most of the time Dropbox will be play well with other services, but there is a known issue with Dropbox that can cause it to block file watching. If this happens for you then simply moving the project usually fixes the issue.
