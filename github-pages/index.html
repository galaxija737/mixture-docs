GitHub Pages

Push and host your project on GitHub pages.

Introduction

GitHub pages settings have changed – you no longer add settings to the mixture settings file. Settings are now all added to a file called .githubsettings in the root of the project, please read on for more details.

Getting Started

Before you can push your project you will need a GitHub account and a repository you would like Mixture to publish to. You can use an existing repository or create a new (empty) one.

When Mixture publishes it will create (if it doesn't exist already) an orphaned branch. Please be aware that when Mixture pushes to this branch it will do a forced push - the branch is configurable.

GitHub Settings

All settings should be added to .githubsettings file in the root of your project, example...

{
  "username": "my-username",
  "password": "my-password",
  "email": "my-email",
  "name": "my-name",
  "gitHubPages": {
  	"repoUrl": "https://github.com/username/repo-name.git",
  	"guid": "self generating guid",
  	"branch": "gh-pages"
  }
}
  • username GitHub Username.
  • password GitHub Token (or password) *.
  • name Your Name.
  • email Your Email.
  • gitHubPages github pages specific details, see below.

gitHubPages elements:

  • repoUrl GitHub repository URL.
  • guid An automatically generated unique identifier.
  • branch default: gh-pages.

* Username and Password are required, however we strongly recommend that you do not use your GitHub password - instead you can generate a Key from your GitHub account. GitHub.com > Settings > Applications > Personal Access Tokens > Create a new token and then use this token in the password field.

As a precaution as soon as you enter your password and save the file Mixture will automatically update it with an encrypted version unique to the application and your account.

By adding the GitHub settings as a new standalone file it can be git ignored from your project if you are using Git, it's also useful if you are working in a team with the project.

Pushing to GitHub Pages

Once all your settings are in place, it's very simple to publish to GitHub Pages with Mixture...

  • Start / Open your project with Mixture
  • Select "Tools"
  • Select "Convert to HTML" (before you publish to GitHub pages remember to convert first - if there are changes to the project)
  • Select "GitHub Pages"
  • Relax. Mixture will push and inform you if any issues.

Please note: It can take 10 minutes once published for GitHub to make your changes live on their URL.
