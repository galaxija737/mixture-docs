Introduction

We've designed in features to aid your development process and optimise by default.

Image optimisation

Mixture will automatically optimise & compress images added to the project by default (currently this works against Jpeg, Jpg, Gif and Png file types).

Optimisation is handled by OptiPNG and JPEGTran.

Script linting

You can manage this feature via the Mixture settings. When this feature is active Mixture will automatically lint scripts when added or updated.

The output will be created in a new js file with an additional "debug" extension, eg my-script.js will result in an output script my-script.debug.js in the same location as the original.

Mixture will only lint files within your project if they are defined within the "preprocessorLocations" setting. Please see the Mixture section and settings.

Debug

You can manage this feature via the Mixture settings.

When in debug mode some features in Mixture will be adjusted:

  • Add debug details to Sass, Less & Stylus files.
  • Set the "mixture.debug" model to true.
  • If you use the "asset_url" in Liquid or Haml and reference a js or css file Mixture will automatically use the minified (.min.) version if it exists - when in debug mode this automatic use of the minified version will be stopped.

We have a small chrome extension which we will be releasing very soon which uses the debug information to report the original source file of a style, you can read more about it here... Mixture + Sass + Less + Chrome.

Minify

By default Mixture will create minified versions of all scripts and stylesheets.

A minified version of the file is output with .min. appended to the name.

If you are using (and we recommend that you do) asset_url with stylesheet_tag or script_tag within your Liquid templates Mixture will automatically detect and serve the minified version. This can be disabled by putting the project into debug mode.

If you wish to maintain comments in minified Javascript files then you can use the JSDoc-style comments that contain "@license" or "@preserve" for Stylesheets you can use /*! comment */.

Note: the live environment will always use the minified version if available.

If mixture encounters a file with .min. or .debug. it will be ignored. This is useful if you are adding files that have already been minified and do not want them processed again.

Mixture will only minify files within your project if they are defined within the "preprocessorLocations" setting. Please see the Mixture section and settings.

Combine

We enable script and style file combination using a custom Mix type, please see the Mix section of preprocessors.
