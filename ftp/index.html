Introduction

FTP settings have changed – you no longer add settings to the mixture settings file. Settings are now added to a file called .ftpsettings in the root of the project, please read on for more details.

Mixture supports FTP for both Mixture and Simple mode projects. The following details the implementations and logic involved.

Mixture Project

Mixture will FTP the "Convert to HTML" output to the remote location you set in the FTP settings within your projects mixture.json settings file.

Only files that have changed will be published and Mixture will not remove any files, but it will overwrite matching.

Any ignore settings in your mixture.json will also be ignored for FTP.

Unlike the Mixture publish for subscription users, the FTP option is purely static and some of the advanced features offered via published projects will not be available.

Simple Mode Project

Mixture will FTP the whole working directory, excluding hidden files and folders and Mixture specific files to the remote location you set in the FTP settings within your projects mixture.json settings file.

Only files that have changed will be published and Mixture will not remove any files, but it will overwrite matching.

Any ignore settings in your mixture.json will also be ignored for FTP.

An example use case for Simple Mode and FTP would be a Wordpress site, we recommend you run Mixture on the themes or theme directory, when you FTP you should set the path to the themes or theme directory at the remote location - Mixture will then allow you to quickly publish changes to your live site.

Settings

Overview of settings:

  • host the hostname or ip address
  • port the port (default is 21)
  • username your username
  • password your password
  • path the remote path (default is the root)
  • secure SSL / TSL support (true or false)

Example settings (which should be added to a file called .ftpsettings in the root of the project):

{
  "host": "ip/hostname",
  "port": 21,
  "username": "my-username",
  "password": "my-password",
  "path": "my-remote-folder",
  "secure": false
}
